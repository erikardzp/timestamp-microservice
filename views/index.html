Timestamp

Es un pequeño microservicio que trabaja de la siguiente manera:

  1. Si se manda una solicitud a /api/:date? con una fecha válida devuelve un objeto JSON con una unix-key que es una marca de tiempo de Unix de la fecha de entrada en milisegundos y una utc-key que es una cadena de la fecha de entrada, ejemplo:

    { "unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT" }

  2. Si se manda una solicitud a /api/1451001600000 siendo el segundo parametro un timestamp regresa un objeto JSON:

    3. { unix: 1451001600000, utc: "Fri, 25 Dec 2015 00:00:00 GMT" }

Request Header Parser

Es un pequeño microservicio que trabaja de la siguiente manera:

  1. Si se manda una solicitud a /api/whoami devuelve un objeto JSON con su dirección IP en la ipaddress-key, con su idioma preferido en la language-key y con su software en la software-key, ejemplo:

{ "ipaddress":"::ffff:159.20.14.100", "language":"en-US,en;q=0.5", "software": "Mozilla/5.0 (X11; Ubuntu; Linux x86_64; rv:50.0) Gecko/20100101 Firefox/50.0" }

URL Shortener

Es un pequeño microservicio que trabaja de la siguiemte manera:

  1. Se puede PUBLICAR una URL y el endpoint api/shorturl/ regresara un objeto JSON con propiedades original_url y short_url, ejemplo:

    2. { original_url : 'https://freeCodeCamp.org', short_url : 1}

  2. Cuando se visite /api/shorturl/:short_url, será redirigido a la URL original.
  3. Si se manda una URL no válida que no sigue el http://www.example.comformato válido, la respuesta JSON contendrá:

    4. { error: 'invalid url' }

Exercise Tracker

Es un pequeño microservicio que trabaja de la siguiente manera, si se manda una solicitud POST /api/users devuelve un objeto JSON:

{ "username": "John Doe", "_id": "619db23047b9882d24aa6fb0" }

  • Puede realizar una solicitud GET /api/users para obtener una lista de todos los usuarios.
  • Puede realizar una solicitud POST /api/users/:_id/exercises con los datos del formulario description, duration y, opcionalmente date. Si no se proporciona ninguna fecha, se utilizará la fecha actual. La respuesta devuelta será el objeto de usuario con los campos de ejercicio agregados:

    • { "username": "John Doe", "description": "Comer", "duration": 45, "_id": "619db23047b9882d24aa6fb0", "date": "Tue Nov 23 2021" }

  • Una solicitud al registro de GET /api/users/:_id/logs devuelve un objeto de usuario con un count propiedad que representa el número de ejercicios que pertenecen a ese usuario.
  • Puede añadir from, to y limit parámetros a una GET /api/users/:_id/logs petición para recuperar parte del registro de cualquier usuario.
    • From y to son fechas en YYYY-MM-DD formato.
    • Limit es un número entero de cuántos registros se deben devolver.
